# Topic bank

One file per content pillar. Each file holds angles, hooks, and proof points so we're never starting from a blank page.

## Suggested pillars (rename / replace as Akila decides)

- `procurement-truths.md` — operator-first hot takes about how procurement actually works in hospitality
- `gtm-built-with-ai.md` — building a GTM function with AI in the loop, lessons in real time
- `purchaseplus-customer-stories.md` — named/anonymized customer wins, with permission status tracked
- `acquisition-journey.md` — Stratton acquisition + 5yr 5x story, only the parts safe to share publicly
- `category-shifts.md` — what's changing in hospitality tech / procurement / supplier dynamics

## File template

```
# [Pillar name]

## Why this pillar
1-2 lines on what audience this pillar is for and what we're trying to be known for.

## Angles
- Bullet list of specific takes within the pillar. Each one is post-sized.

## Hooks (recyclable openers)
- "Most hotel GMs don't trust procurement software. Here's why they're right."
- "The number nobody puts in the procurement pitch deck."

## Proof points (facts/anecdotes available)
- Bundle integration result: ___
- Stratton acquisition rationale: ___
- 14-property hotel group rollout: ___

## Off-limits
What NOT to write about in this pillar (sensitive, NDA, half-baked).
```
